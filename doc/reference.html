1. Introduction

MSIM is a light-weight computer simulator based on MIPS R4000. It is used for education and research purposes, mainly to teach the construction and implementation of operating systems. MSIM is distributed with source code under the GNU GPL license to make possible modifications for users and works on most POSIX-compliance environments (mainly Linux, Mac OS X, but can be also compiled in Cygwin or MinGW in Windows).

MSIM provides following features (along others):

  • MIPS R4000 CPU
    • full memory management (TLB)
    • multiprocessor support
    • the instruction set is restricted to 32 bits
    • CPU cache is not simulated
  • simple debugging features (including disassembling, register content dump)
  • several simple hardware devices
  • various hardware-manipulating commands
  • widely configurable memory mapping of devices
  • script-like start-up configuration file

MSIM does not aim to be a speed-optimized real hardware simulator, but rather a fully deterministic simulator useful for kernel debugging. There are several other projects which aim speed-optimized simulation (e.g. GXemul), but they are more complex to use.

Other hardware devices are designed in a very straightforward way to emphatize basic principles of operation, but do not confuse the programmers with historical heritage and complex issues.

There are several modes the MSIM runs in. Typically, the user will simply configure the system and execute the code. For debugging purposes, the simulation can be switched into trace mode, where all executed instructions are displayed together with system events. In interactive mode the user communicates with the simulator via a command line interface and can modify the environment of the running system.

3. Command line parameters

3.1. Version -V, --version

Displays MSIM version information and quits.

Syntax: -V|--version

Example

~$ ./MSIM -VEnter
MSIM version 1.3 Copyright (c) 2000-2006 Viliam Holub

3.2. Configuration file -c, --config

Specifies the configuration file name which overrides the default searching rules.

Syntax: -c|--config[=]filename

Example

~$ MSIM -c my.conf

3.3. Interactive mode -i, --interactive

The simulator enters the interactive mode immediately after the configuration file has been processed.

Syntax: -i|--interactive

Example

~$ MSIM -iEnter
[MSIM]

3.4. Trace mode -t, --trace

Enters the trace mode, but does not enable the interactive mode.

Syntax: -t|--trace

Example

~$ MSIM -tEnter
    BFC00000    lui   sp, 0x8000        # 0x8000=32768, sp: 0x0->0x80000000
    BFC00004    addiu sp, sp, 0x7000    # 0x7000=28672, 
					# sp: 0x80000000->0x80007000

    80400000    jal   +0x400f4c         # 0x400f4c=4198220, 
 					# ra: 0x0->0x80400008
    80400004    nop
...

3.5. GDB mode -g, --remote-gdb

Enters the gdb mode. The gdb mode is rather experimental in version 1.3.

Syntax: -g|--remote-gdb[=]port_number

3.6. Help -h, --help

Prints a help and quits.

Syntax: -h|--help

Adds a new device into the system.

Removes the device from the system.

Exits MSIM.
md
Dumps words from unmapped memory.

Dumps instructions from unmapped memory.

Prints user message.

Go to the specified address or continue.

Simulates one or a specified number of instructions.

Sets environment variables.
unset
Unsets environment variables.
help
Displays a help text.

8.2. Add add

Adds a new device into the system under a specified name.

Syntax: add device_type device_name [device_parameters...]

where:

device_type
specifies a device type. The list of available device types is in the Devices section,
device_name
is a device name. The name must not be equal to a system command and must be unique among already added devices, and
device_parameters
are initial device parameters. The actual parameters depends on the device type.

Example

The first example adds a new printer "p0" to the system. The printer register is located at 0x0001000.

[MSIM] add dprinter p0 0x1000Enter
[MSIM]

The second example adds a read-only memory "m0" mapped at 0x10000. The size of the memory is 16 kilobytes.

[MSIM] add rom m0 0x10000 16kEnter
[MSIM]

8.3. Remove rm

Removes the specified device from the system.

Syntax: rm device_name

where:

device_name
is a name of the device that has to be removed.

Example

The following example removes a device named "Joe" from the system.

[MSIM] rm JoeEnter
[MSIM]

8.13. Step step

Executes one or a specified number of cycles. The step command is default - it is executed if no command is specified.

Snytax: step [count]

where:

count
is the optional number of cycles to execute.

Example

In the following example, the user executes 3 steps on a 2-processor machine:

[MSIM] s 3Enter
 0  80400ACC    mfc0  v0, status        # v0: 0x0->0x8001
 1  80400EC4    srl   a1, a1, 0x18      # 0x18=24, a1: 0x6538b04f->0x65

 1  80400EC8    sll   v0, a1, 0x04      # v0: 0x219dad69->0x650
 0  80400AD4    and   v0, v0, v1        # v0: 0x8001->0x8000
 1  80400ECC    addu  v0, v0, a1        # v0: 0x650->0x6b5
[MSIM]

8.14. Set set

Sets an environment variable on or to a specified value or prints a list of all variables..

Syntax: set variable [= value]

where:

variable
is an environment variable name to set,
value
is an optional value to assign to the environment variable.

Refer to the specialized section for a list and a description of available variables. If value is not specified, the variable is set to the logical true. Note not all variables can hold a logical value. If even variable is not specified, set prints the list of variables and their values.

For boolean variables, there are available default constant for true: true, on, yes and for false: false, on, no and their prefixes.

Example

In the first example, we all the variables:

[MSIM] setEnter
List of all variables:
Disassembling features
	iaddr = on
	iopc = off
	icmt = on
	iregch = on
	ireg = 2
Debugging features
	trace = on
[MSIM]

In the second example, we set the trace mode to on:

[MSIM] sEnter
[MSIM] set traceEnter
[MSIM] sEnter
    80401378    addiu a0, a0, 0x40      # 0x40=64, a0: 0xb7b8->0xf8
[MSIM]

And in the third example, we set the type of register names:

[MSIM] set ireg = 2Enter
[MSIM] id 0x00401330 1Enter
    00401330    srl   v0, v0, 0x01        
[MSIM] set ireg = 0Enter
[MSIM] id 0x00401330 1Enter
    00401330    srl   r2, r2, 0x01
[MSIM]

10. Special instructions

To further improve the experience of debugging the system, MIPS reserves and implements several MSIM-specific instructions for debugging purposes. The form of an instructions allows to change the behavior of the simulator exactly at the place required, with only a minimal influence to the debugged code.

DTRC
Enables the trace mode.
DTRO
Disables the trace mode.
DINT
Enables the interactive mode.
DRV
Writes the content of all general registers on the screen.
DHLT
Halts the simulation.
DVAL
Dumps the a0 general register.

Example

The following example shows an implementation in C:

1 #define ___traceon()    asm volatile ( "\t.word\t0x39\n");
2 #define ___traceoff()   asm volatile ( "\t.word\t0x3d\n");
3 #define ___regview()    asm volatile ( "\t.word\t0x37\n");
4 #define ___halt()       asm volatile ( "\t.word\t0x28\n");
5 #define ___val( i)      asm volatile ( "\t.word\t0x35\n" :: "r" (i));

10.1. Trace on DTRC

Enables the trace mode. Together with DTRO instruction, DTRC is useful to show a part of executed code.

Opcode: 0x39

10.2. Trace off DTRO

Disables the trace mode. Together with DTRC instruction, DTRO is useful to show a part of executed code.

Opcode: 0x3d

10.3. Interactive mode on DINT

Enables the interactive mode. The command line is activated immediately.

Opcode: 0x29

10.4. Register view DRV

The instruction writes the content of all the general register on the output.

Opcode: 0x37

10.5. Halt machine DHLT

Halts the machine as well as the simulation immediately. Useful also as the power off feature.

Opcode: 0x28

10.6. View value DVAL

Prints the content of the a0 (r4) register. The instruction can be used to instantly print a value of a variable.

Opcode: 0x35
